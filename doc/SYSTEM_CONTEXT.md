**Project Context (项目背景):**
你正在维护一个高性能、线程安全的 C++ 任务执行系统 (`TaskSystem`)。该系统采用 CMake 构建，包含 GoogleTest 单元测试，设计上实现了逻辑与执行的分离。

**System Architecture (系统架构):**

1.  **`ThreadPool` (核心执行引擎)**:
    *   **动态伸缩**: 支持基于配置的动态扩容（队列长度或等待时间策略）。
    *   **优先级调度**: 内部使用最大堆实现优先级队列。
    *   **任务窃取**: 每个线程拥有本地队列，支持 LIFO 本地获取和 FIFO 远程窃取。为了防止优先级反转，仅 NORMAL 任务进入本地队列。

2.  **`TaskExecutor` (高层接口)**:
    *   **泛型任务链**: 返回 `TaskHandle<T>`，支持通过 `.then()` 传递数据。
    *   **异常传播**: 自动捕获并传播 `std::exception_ptr`，支持 `.on_error()` 捕获。
    *   **任务聚合**: 支持 `when_all()` 和 `when_any()`。
    *   **调试增强**: 通过 `TASK_FROM_HERE` 捕获源码位置，自动监控长任务。

3.  **`Logger` (基础设施)**:
    *   线程安全的单例日志模块。
